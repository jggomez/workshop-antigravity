# Slides

This directory contains the core visual materials for the workshop. The progression of these decks is designed to shift the developer’s mindset: from a manual artisan to a **System Architect and AI Orchestrator**.

### 1. The Vibe Coding Paradox: Mastery & Risks
**File:** `Vibe-Coding-Paradox.pdf`

* **Description**: An exploration of "Vibe Coding," defined as building software by describing desires in natural language to AI tools.
* **Importance**: This is the "wake-up call" regarding the **Vibe Coding Hangover**. It highlights critical security risks, noting that 170 out of 1,645 projects on a popular vibe platform exposed raw databases and personal data to the public internet.

### 2. Spec-Driven Development (SDD)
**File:** `SDD-Framework.pdf`

* **Description**: Presents SDD as an evolution of TDD and BDD optimized for AI Coding Agents. It replaces "dead text" documents with structured files (Markdown, YAML) that explicitly describe user journeys, tech stacks, and success criteria.
* **Importance**: It establishes a rigorous workflow to prevent "hallucinations and drift". The process moves through four distinct phases: **Specify** (defining the problem), **Plan** (architectural decisions), **Break Down** (atomic tasks), and **Implement/Verify**.

### 3. Vibe Engineering: From Chaos to Production
**File:** `Mastering-Vibe-Engineering.pdf`

* **Description**: A framework for orchestrating LLMs with rigorous specs, automated testing, and strict human oversight[cite: 236]. [cite_start]It details the shift in the developer's role from writing syntax (80% of traditional work) to **Code Review & Verification** (50% of the Vibe Engineering workflow)[cite: 262, 268, 269].
* **Importance**: It teaches how to find the **"Goldilocks Zone"** of software creation. It provides tactical guardrails, such as using project-specific rules and following a strict **"Prompt → Review → Refactor"** cycle.

## Learning Summary

| Concept | Approach | Result |
| :--- | :--- | :--- |
| **Vibe Coding** | Natural Language & "Prompt and Pray" | Fast prototyping, but brittle systems and massive technical debt. |
| **SDD** | Structured Specs & Success Criteria | Deterministic AI behavior and maintainable code. |
| **Vibe Engineering** | Architecture, Guardrails, & Human-in-the-Loop | Scalable, secure, and production-ready software. |

## The Three Pillars of Mastery
To succeed in this workshop, students must internalize these pillars:
1.  **Vision**: Start with architectural clarity and a blueprint, not a vague idea.
2.  **Process**: Adopt a rigorous, iterative workflow (Plan → Implement → Review).
3.  **Guardrails**: Impose explicit constraints on the AI to prevent technical drift.
