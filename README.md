# Workshop: Spec-Driven Development & Vibe Engineering with Google Antigravity

## Description
This technical workshop is designed for developers and architects aiming to master the **Agentic AI-First Era**. Through a hands-on approach, we will explore how to use **Google Antigravity** to transform ideas into functional software using rigorous specifications and *Vibe Engineering*—a development technique based on intuitive iteration and semantic refinement with AI agents.

## Objectives
1.  **Understand SDD (Spec-Driven Development):** Learn to draft specifications that AI agents can execute with 95%+ precision.
2.  **Agentic Architecture:** Configure the Antigravity environment to maximize the development agent's autonomy.
3.  **Vibe Engineering:** Iterate on design and user experience using natural language and continuous feedback loops.
4.  **End-to-End Implementation:** Build a functional application from initial definition to assisted deployment.

## Workshop Structure (Step-by-Step)

| Step | Directory | Description |
| :--- | :--- | :--- |
| **01** | `01-slides` | Visual materials and theoretical foundations of SDD and Antigravity architecture. |
| **02** | `02-videos` | Pre-recorded demos of critical flows and *Agentic Workflow* concepts. |
| **03** | `03-spec-app` | Defining **Business Requirements** and creating the initial `spec.md` to guide the AI. |
| **04** | `04-design-app` | Solution modeling: Initial C4 diagrams and API contract definitions. |
| **05** | `05-antigravity-config` | Configuring the `.antigravityrc` file, defining style rules, and agent permissions. |
| **06** | `06-vibe-engineering` | Aesthetic and functional refinement phase ("The Vibe") via interactive prompts and hot-fixes. |

## Tools & Prerequisites

### Required Software
* **Google Antigravity IDE** (Agentic-native environment).
* **Gemini** (Underlying LLM models for reasoning).
* **Node.js v22+** or **Python 3.12+** (Application stack).
* **Git** (Version control for code and specs).

### Prior Knowledge
* Basic understanding of **Large Language Models (LLMs)**.
* Familiarity with architecture patterns (**Microservices, Clean Architecture**).
* Previous experience using **Terminal/CLI**.
